National public view

The national public view will define the planning application data that should be published openly from the wider planning application data standard.

The view is being developed as an explicit extraction from the standard. Nothing enters the national public view unless the standard says it should be included.

What the view will define

The national public view is expected to be defined by a future national-public-view.schema.md file.

That schema will specify which datasets and fields are included in the public view. Some datasets may include only selected records.

Expected inclusion rules

  • a dataset is in the national public view only if it appears in the public view schema
  • a field is in the national public view only if it is listed for that dataset
  • document records are expected to use public-register-status to decide whether they are included
  • timeline records may use codelist profile rules to decide which event types are included

What is out of scope

The national public view will define conformant public output. It will not prescribe supplier back-office architecture or local storage design.

Local publication and redaction processes remain separate decisions unless the standard explicitly defines a public-view inclusion rule.
