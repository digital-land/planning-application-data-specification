module: materials

Materials

What materials are being used for the proposed development

Fields

Proposal material details
proposal-material-details

Whether the proposal involves material details that need to be provided

Required: True

Datatype: boolean

Expected number: 1

Building elements
building-elements

Details of materials for a specific building element such as walls, roof, windows or doors

Required: False

Datatype: object

Expected number: 1 or more

Building element component

Building element type
building-element-type

The part of building the materials relate to, such as walls, roofs, windows, or doors

Required: True

Datatype: enum

Codelist: building-element-type

Expected number: 1

Existing materials
existing-materials

Description of the materials currently used for this building element

Required: False

Datatype: string

Expected number: 1

Proposed materials
proposed-materials

Description of the materials proposed for this building element as part of the development

Required: False

Datatype: string

Expected number: 1

Materials not known
materials-not-known

Indicates the materials for this building element are not yet known

Datatype: boolean

Expected number: 1

References to supporting documents that have been uploaded with the application

Required: False

Datatype: object

Expected number: 1 or more

Supporting document component

Reference
reference

A unique reference for the data item

Required: True

Datatype: string

Expected number: 1

Details
details

Additional details or information about an item

Required: False

Datatype: string

Expected number: 1

Rules

  • Each building-element must have a unique building-element-type
  • At least one of: existing-materials, proposed-materials or materials-not-known must be provided for each building-element
  • materials-not-known cannot be true if existing-materials or proposed-materials is provided
  • supporting-documents must reference valid documents in the application
