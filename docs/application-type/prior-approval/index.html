Prior approval

  • Details
  • Fields
  • Modules
  • Feedback

This applies to developments with permitted development rights (where developments are granted planning permission by national legislation without the need to submit a planning application)

Details

This is a base-type schema. It defines common fields and modules that other application types can extend. It should not be used as a standalone application type.

Application
prior-approval
Base type
true

2025-01-07
Legislation
  • https://www.legislation.gov.uk/uksi/2015/596/contents

Fields

These fields are captured at application level before the application-specific modules.

Submission details

Details about the submitted payload, including how it is identified, routed, validated, handled and traced

Reference: submission-details

Required: True

Modules

This application uses the following modules. Each module groups fields that collect a related set of information.

  • Agent contact details (agent-contact)

    Name and contact information if an agent is being used.

  • Agent details (agent-details)

    Name and contact information if an agent is being used.

  • Applicant contact details (applicant-contact)

    Telephone number and email address of the applicant.

  • Applicant details (applicant-details)

    Name and contact information for the parties making the application.

  • Conflict of interest (conflict-of-interest)

    Details of any conflict of interest that may exist between the applicant and planning authority.

  • Checklist (checklist)

    Checking whether all the requirements of the form have been met, such as proof of payment or supporting documentation.

  • Declaration (declaration)

    Signed and dated verification of the application's accuracy.

  • Site details (site-details)

    Where the proposed development will be built.
