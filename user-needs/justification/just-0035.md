The need to see the full post-submission timeline is partly met when each timeline event can record what happened, when it happened, and optional notes that explain the context of the event.

The notes field helps users interpret repeated events, delays, route changes or other process events where the event type and date alone may not be enough. It should be used for human-readable context, not as a substitute for standard event types or other structured fields where those are needed.
